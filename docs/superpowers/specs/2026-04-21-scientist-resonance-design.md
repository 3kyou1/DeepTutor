# Scientist Resonance 设计文档

## 1. 背景与目标

DeepTutor 当前已经具备两类与用户画像相关的基础能力：

- 基于 CoPA 因子的长期回答偏好画像；
- 基于冷启动问卷生成首版个性化信息。

现计划在此基础上新增一个偏趣味、可讨论、具人物感的画像模块：`Scientist Resonance`。该模块不追求心理测量式精确分类，而是基于用户现有 profile，为用户生成一个“你像哪类科学家/研究者”的人物镜像。

该功能的核心价值不是“测得准”，而是：

- 像得妙：能让用户觉得“这个比喻有意思，而且说得通”；
- 讲得活：输出具备人物钩子、研究气质和讨论空间；
- 能持续：既能给长期主原型，也能表达最近阶段状态。

## 2. 产品定位

### 2.1 核心定位

该功能是一个“有解释的趣味映射”，而不是：

- 严格的人格测量系统；
- 学术能力评估工具；
- 命运式标签系统；
- 对用户做“天才/非天才”判断的装置。

系统应表达的是：

> 你在某些思维方式、研究气质和人格张力上，让人联想到某位研究者。

而不是：

> 你就是某某科学家，或者你具备与其等同的能力与人生轨迹。

### 2.2 首发入口

首发版本通过两个入口呈现：

1. **冷启动完成后自动生成**
   - 在冷启动问卷完成、首版 CoPA profile 建立后，自动生成第一版长期主原型卡。
2. **Profile 页面长期展示**
   - 将 `Scientist Resonance` 作为长期画像的一部分展示在 profile 页面中。

不作为首发重点的入口：

- 聊天中临时触发；
- 独立测试页；
- 开放式自由匹配。

## 3. 设计原则

### 3.1 趣味性优先于伪精确

输出不使用百分比、精细打分或“83.4% 像某人”这类表达，避免制造不存在的测量精度。

推荐使用如下置信度风格：

- 强共振；
- 偏某一侧面的共振；
- 当前阶段性共振。

### 3.2 思维方式为主轴

匹配主轴是“研究型思维方式”，人格气质是强辅助信号，学习与表达偏好是弱辅助信号。

### 3.3 长期稳定 + 近期漂移

功能必须同时支持两层视角：

- **长期主原型**：稳定底色，不应频繁抖动；
- **最近状态原型**：反映近期聊天中的阶段性倾向，可轻度漂移。

### 3.4 人物感强，但不过度神化

文案应具备故事感、辨识度与研究气质，但不能神化、病理化或宿命化人物。

## 4. 用户体验

### 4.1 冷启动后的体验

当用户完成冷启动问卷后：

1. 系统先生成/刷新 CoPA 画像；
2. 随后基于该画像生成第一版 `Scientist Resonance.long_term`；
3. 在冷启动结果中展示一张“长期主原型卡”；
4. 用户之后可在 Profile 页面反复查看。

### 4.2 Profile 页面体验

Profile 页面中新增一个模块：`Scientist Resonance`。

模块由两部分组成：

- **长期主原型**
  - 1 位主匹配人物；
  - 2 位备选侧面；
  - 2-4 个共振维度标签；
  - 一段带人物感的解释文案。

- **最近状态原型**
  - 1 位近期更像的人物；
  - 可选 1-2 个近期共振侧面；
  - 一段说明“为什么是最近阶段像他”的短文案。

### 4.3 期望用户感受

用户看到后应产生的反应是：

- “这个比喻挺妙的。”
- “不是说我就是他，但确实像他的某一面。”
- “我愿意继续追问为什么像这个人，而不是另一个人。”

## 5. 候选人物库设计

### 5.1 候选池策略

采用**精选固定库**，不采用完全开放候选池。

原因：

- 保证人物质量、辨识度和风格一致性；
- 降低 LLM 随机匹配冷门人物的风险；
- 便于控制文案调性和结果稳定性；
- 便于后续扩展人物标签体系。

### 5.2 初始规模

首版建议维护 `20-40` 位研究者。

该规模足以：

- 保证人物有明显差异；
- 覆盖多种研究气质；
- 避免库过小导致结果单调；
- 避免库过大导致解释质量变差。

### 5.3 入选标准

入选人物不以“伟大程度”排序，而以以下标准筛选：

- 方法气质鲜明；
- 人物故事或研究风格有辨识度；
- 足够有趣，能承载趣味镜像；
- 具备较强“研究人格”形象；
- 用户即使不熟悉，也能从文案中感受到人物魅力。

### 5.4 覆盖方向

人物池建议覆盖但不限于以下方向：

- 直觉爆发型；
- 结构建构型；
- 反常规探索型；
- 深潜执拗型；
- 跨界表达型；
- 问题驱动型；
- 体系建构型；
- 非标准路径型。

### 5.5 人物结构化字段

每位候选人物建议维护一份结构化 profile，而不是只存名字和简介。

建议字段：

- `name`：人物名；
- `era_or_field`：时代或主要领域；
- `short_hook`：一句人物钩子；
- `core_traits`：3-6 个核心标签；
- `thinking_style`：主要思维方式；
- `temperament`：人格/研究气质；
- `learning_signal`：像该人物的用户通常在学习或提问中会出现的信号；
- `why_fun`：这个人物“有意思”的原因；
- `avoid_overclaim`：避免写得过满的提醒语。

## 6. 匹配逻辑

### 6.1 输入来源

#### 长期主原型

主要输入：

- CoPA 长期画像；
- 冷启动问卷汇总结果；
- 长期 profile 中已有稳定偏好描述。

#### 最近状态原型

主要输入：

- 最近若干轮用户消息；
- 最近阶段任务风格；
- 最近阶段情绪/推进方式信号。

### 6.2 匹配维度

#### 主维度：思维方式

建议抽取的高层信号包括：

- 直觉跳跃；
- 形式严密；
- 模式敏感；
- 构造导向；
- 问题驱动；
- 体系建构；
- 抽象深潜；
- 非标准路径偏好。

#### 强辅助维度：人格气质

建议抽取的高层信号包括：

- 独行 / 合作；
- 执拗 / 灵动；
- 克制 / 外放；
- 深潜 / 广撒；
- 传奇感 / 稳定感；
- 自成体系 / 强互动型。

#### 弱辅助维度：学习与表达偏好

建议抽取的高层信号包括：

- 先抓本质还是先看细节；
- 先直觉试探还是先组织框架；
- 偏个人理解还是偏标准表述；
- 偏探索路径还是偏结果收敛。

### 6.3 匹配方式

不采用“纯分数排行榜”式输出。

推荐采用两阶段启发式匹配：

1. **特征提取阶段**
   - 从长期画像或近期消息中提取 5-8 个高层特征标签；
   - 形成简明的用户当前研究气质摘要。
2. **人物共振阶段**
   - 在固定人物库中寻找 1 位整体共振最强的人物作为主匹配；
   - 再找 2 位共享不同侧面的备选人物；
   - 备选人物不等价于“分数第二、第三”，而是表示相邻但不同的共振面。

### 6.4 主匹配与备选的语义区分

#### 主匹配

表达的是：

- 用户最稳定、最集中、最有代表性的研究者镜像。

#### 备选

表达的是：

- 用户在某一特定维度上也像另一位人物；
- 这些人物用于补足“不是单一标签”的丰富度。

例如：

- 主匹配偏“直觉爆发 + 非标准路径”；
- 备选 A 偏“问题驱动 + 自我解释”；
- 备选 B 偏“抽象深潜 + 自成体系”。

## 7. 数据模型

### 7.1 存储策略

不新开独立 memory 文件，直接在现有 `PROFILE.md` 中增加受控区块，例如：

- `## Scientist Resonance`

原因：

- 与现有 profile 体系一致；
- 便于 Profile 页面直接渲染；
- 便于复用当前 memory/profile 更新机制；
- 避免产生过多分散的共享状态文件。

### 7.2 结构化结果建议

内部建议先生成结构化 JSON，再渲染到 `PROFILE.md`。

建议结构：

```json
{
  "long_term": {
    "primary_match": {
      "name": "string",
      "hook": "string",
      "confidence_style": "strong_resonance"
    },
    "alternatives": [
      {
        "name": "string",
        "reason": "string"
      }
    ],
    "resonance_axes": ["string"],
    "narrative": "string"
  },
  "recent_state": {
    "primary_match": {
      "name": "string",
      "hook": "string",
      "confidence_style": "phase_resonance"
    },
    "alternatives": [
      {
        "name": "string",
        "reason": "string"
      }
    ],
    "resonance_axes": ["string"],
    "narrative": "string",
    "based_on_window": 12
  }
}
```

说明：

- `long_term`：长期稳定人物镜像；
- `recent_state`：近期阶段人物镜像；
- `primary_match`：主匹配人物；
- `alternatives`：2 个备选侧面；
- `resonance_axes`：2-4 个共振标签；
- `narrative`：具人物感的解释文案；
- `confidence_style`：置信风格，不用数值。

### 7.3 PROFILE.md 渲染建议

渲染为结构清晰但有文学性的 markdown，而不是“测试报告”。

建议格式：

```md
## Scientist Resonance

### 长期主原型
- 主匹配：Srinivasa Ramanujan
- 共振类型：强共振
- 共振维度：直觉跳跃、非标准路径、模式敏感
- 人物钩子：像是在未写全推导前就已先看见结构的人。

你长期更像拉马努金式的研究者气质：不是因为“神秘”，而是因为你更容易先捕捉到模式和结构，再回头为这些直觉寻找可表达的形式。

备选侧面：
- Richard Feynman：你也有很强的问题驱动与个人化理解倾向。
- Alexander Grothendieck：你在抽象深潜与自成体系这一面也与他共振。

### 最近状态原型
- 主匹配：Richard Feynman
- 共振类型：当前阶段性共振
- 共振维度：问题驱动、解释冲动、推进感

最近这段时间，你更像费曼式状态：不是停在抽象欣赏，而是不断想把一个东西“弄懂到能亲手讲明白”。
```

## 8. 生成与更新策略

### 8.1 首次生成

首次生成时机：

- 冷启动问卷完成；
- CoPA 首版画像已落盘；
- 可立即基于该结果生成 `long_term`。

首版可先只生成：

- `long_term.primary_match`
- `long_term.alternatives`
- `long_term.resonance_axes`
- `long_term.narrative`

`recent_state` 可在有足够近期消息后再补齐。

### 8.2 周期更新

#### 长期主原型更新

低频更新。只有在稳定画像明显变化时才允许切换主匹配。

要求：

- 不应今天是 A，明天变 B；
- 若确需切换，应能解释为“主原型逐渐向某方向迁移”；
- 更适合设置为阈值触发而非每轮刷新。

#### 最近状态原型更新

高频更新，但漂移范围要受控。

建议：

- 基于最近固定窗口的用户消息；
- 允许随当前研究/学习状态变化；
- 文案必须明确指出这是“最近阶段”的像法。

### 8.3 输出稳定性策略

为避免随机感，建议：

- 使用固定人物池；
- 限制每次只在相近人物簇中切换；
- 对长期卡设置较高变更门槛；
- 对近期卡设置较强“近期证据”要求；
- 输出时保留“像的是哪一面”而不是绝对断言。

## 9. 文案风格要求

### 9.1 推荐风格

文案应接近“研究者人物评论”：

- 有判断；
- 有人物气味；
- 有节制；
- 不夸张；
- 不像心理测试页面。

### 9.2 避免风格

避免如下表达：

- “你就是某某转世”；
- “你拥有天才般的……”；
- “你和他一样孤独/悲剧/疯狂”；
- “你有 X% 的某某人格”；
- “你注定适合成为……”

### 9.3 推荐表达模板

建议使用如下句式：

- “你更像他在某一研究侧面的气质，而不是整个人生轨迹。”
- “像的不是传奇叙事本身，而是那种处理问题的路径。”
- “你与他共振的地方，在于……”
- “如果说长期底色像谁，那么你最近这一阵子更像……”

## 10. 风险与约束

### 10.1 神化风险

如果文案过度拔高，容易让用户把它理解为能力认证。

控制方式：

- 明确是“共振”而非“认证”；
- 重点写思维方式，不写能力等级；
- 不使用英雄化语言。

### 10.2 病理化和悲剧叙事风险

部分历史人物带有复杂私人经历，不适合直接映射给用户。

控制方式：

- 不使用精神状态、社会关系、悲剧命运作为匹配理由；
- 只围绕研究气质和思维路径写作。

### 10.3 人物冷门风险

如果候选池太开放，匹配结果可能落到用户完全陌生的人物上。

控制方式：

- 首版使用精选固定库；
- 每个人物都提供一句钩子，帮助用户快速感知人物气质。

### 10.4 结果漂移风险

如果长期卡频繁变化，用户会觉得系统在随机抽卡。

控制方式：

- 长期卡低频更新；
- 近期卡承担主要波动；
- 明确区分“长期底色”和“最近状态”。

## 11. 实现建议

### 11.1 模块边界

建议新增独立服务层，例如：

- `deeptutor/services/personalization/scientist_resonance.py`

职责：

- 读取 CoPA profile；
- 读取最近消息窗口；
- 读取/维护候选人物库；
- 生成结构化 resonance 结果；
- 渲染并写回 `PROFILE.md` 的受控区块。

### 11.2 候选库位置

候选人物库建议独立维护，例如：

- `deeptutor/services/personalization/scientist_candidates.py`
- 或 `deeptutor/services/personalization/data/scientists.yaml`

要求：

- 可读、可维护；
- 便于后续手工润色人物标签；
- 便于测试中加载固定数据。

### 11.3 与现有流程的关系

建议与现有个性化流程并行但分层：

- CoPA：回答偏好画像；
- Scientist Resonance：研究者镜像画像。

两者关系：

- 前者提供“用户喜欢怎样被回答”；
- 后者提供“用户像哪类研究者原型”；
- 两者都写入 `PROFILE.md`，但使用不同受控区块。

## 12. 测试建议

首版至少覆盖以下测试：

- 候选人物库加载与 schema 校验；
- 长期 profile 到共振标签的提取稳定性；
- 主匹配 + 2 备选输出结构完整性；
- `PROFILE.md` 指定区块写入不破坏其他内容；
- 长期卡更新门槛控制；
- 近期卡基于最近窗口的漂移逻辑；
- 文案字段缺失时的降级策略。

## 13. 非目标

首版明确不做：

- 精确量化相似度百分比；
- 用户自定义任意人物匹配；
- 开放式网络检索人物；
- 基于该人物原型直接驱动整套聊天人格；
- 社交分享海报或视觉化风格页。

## 14. 最终定义

`Scientist Resonance` 是一个建立在现有 profile 之上的人物镜像模块。它通过精选固定库，将用户的长期研究气质与近期阶段状态分别映射为“长期主原型”和“最近状态原型”，并以 `1 个主匹配 + 2 个备选侧面` 的方式呈现。其输出强调思维方式与人格张力的共振，采用有故事感但不过度神化的文案，服务于冷启动结果和 Profile 页面展示。
